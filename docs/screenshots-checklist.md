# Screenshots / Proof Checklist

Capture and place these in the `screenshots/` folder before your final GitHub push:

- [ ] Project folder structure (IDE sidebar or `tree` command output)
- [ ] User registration console output
- [ ] User login console output
- [ ] Complaint submission flow (title/description/category/priority prompts)
- [ ] Generated complaint ID shown after submission
- [ ] User's "View my complaints" / "Track by ID" screen
- [ ] Admin dashboard menu
- [ ] Admin "assign complaint" output
- [ ] Status update output (e.g. moving to IN_PROGRESS)
- [ ] Resolution note output (complaint becomes RESOLVED)
- [ ] Complaint history / full complaint printout (all fields visible)
- [ ] Feedback submission output
- [ ] Saved data file contents (`cat data/complaints.txt` in terminal)
- [ ] GitHub repository homepage preview
- [ ] Rendered README.md preview on GitHub
